# Cold Email Outreach FR

## Principes fondamentaux

Le cold email FR n'est pas du cold email US traduit.
Différences clés :
- Ton direct mais pas agressif (pas de "Hey First Name!")
- Objet court et factuel (pas de clickbait)
- Corps court : 3-4 phrases max pour l'accroche
- RGPD : mention obligatoire du droit de désinscription
- Pas de fausse familiarité (pas de "j'espère que tu vas bien")
- Heure d'envoi optimale : mardi-jeudi 8h30-9h30 ou 14h-15h

## Structure email type

**Objet** : [Bénéfice concret] pour [métier/secteur]
Exemple : "Automatiser les relances clients pour les cabinets comptables"

**Corps** :
1. Accroche : observation spécifique sur leur secteur (1 phrase)
2. Problème : douleur identifiée (1 phrase)
3. Solution : ce qu'on fait concrètement (1-2 phrases)
4. CTA : une seule question fermée ou proposition de créneau
5. Désinscription : "Si ce sujet ne vous concerne pas, répondez 'stop'."

## Templates par verticale

### Comptables / Experts-comptables
```

Objet : Automatiser les relances impayés pour les cabinets comptables

Bonjour [Prénom],

Les cabinets comptables perdent en moyenne 3h/semaine en relances manuelles de paiement client.

Nous avons développé un agent IA qui gère ces relances automatiquement, en respectant le ton de votre cabinet et les délais légaux.

15 minutes cette semaine pour vous montrer comment ça fonctionne sur vos données ?

Cordialement,

[Signature]

Si ce sujet ne vous concerne pas, répondez 'stop'.

```

## Séquence type (3 touchpoints max)

**Email 1 (J0)** : Template principal ci-dessus
**Email 2 (J+3, si pas de réponse)** : Relance courte
```

Bonjour [Prénom],

Je me permets de revenir vers vous suite à mon email de jeudi.

Avez-vous eu l'occasion d'y jeter un œil ?

[Signature]

```
**Email 3 (J+7, dernier)** : Break-up email
```

Bonjour [Prénom],

Je ne veux pas vous encombrer — c'est mon dernier message sur ce sujet.

Si la question de l'automatisation devient pertinente pour vous dans les prochains mois, n'hésitez pas à revenir vers moi.

Bonne continuation,

[Signature]

```

## RGPD — règles obligatoires

- Toujours inclure une option de désinscription explicite
- Ne pas collecter de données sans consentement
- Supprimer les contacts qui répondent 'stop' dans les 72h
- Conserver les preuves de consentement
- Base légale : intérêt légitime (B2B) — documenter le lien avec l'activité professionnelle du destinataire

## Anti-patterns à éviter

- "J'espère que ce message vous trouve en bonne santé"
- Objets en majuscules ou avec points d'exclamation
- Plus de 150 mots dans le corps
- Plusieurs CTAs dans le même email
- Liens de tracking visibles dans le corps
- Envoyer le vendredi après-midi ou le lundi matin
